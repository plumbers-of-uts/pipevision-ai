# PipeVision Module Contracts

> Generated by Plan Workflow | Session: pipevision-ext-001 | Date: 2026-04-30

This project has **no backend HTTP API** (zero-infrastructure design). The contracts below define data shapes between modules: ONNX model I/O, model metadata, detection results, IndexedDB schema, model loading state, and the HF Spaces fallback API.

---

## C1. Model I/O Contract (ONNX)

### Input
- **Name**: `images` (verify with `session.inputNames` after export)
- **Shape**: `[1, 3, 640, 640]` (NCHW, fixed)
- **Dtype**: `float16`
- **Value range**: `[0, 1]` (normalized RGB)
- **Preprocessing**: Letterbox resize (preserve aspect ratio, padding with 114/255)

### Output
- **Name**: `output0` (verify with `session.outputNames`)
- **Shape**: `[1, 11, 8400]` — `4 (xywh) + 7 (class scores)` × 8400 anchor predictions
  - Note: Ultralytics may output transposed `[1, 8400, 11]` — handle both in `postprocess.ts`
- **Dtype**: `float16`
- **Coordinates**: cx, cy, w, h in normalized 640×640 space
- **NMS**: NOT included in graph (handled in TypeScript, see C3)

### Example session call
```typescript
const feeds: Record<string, Tensor> = { images: tensor };
const results = await session.run(feeds);
const raw = results.output0;  // ort.Tensor with float16 data
```

## C3. Detection Result Contract

Output of `postprocess.ts`, consumed by `DetectionCanvas` and `HistoryRecord.detections`.

```typescript
export interface Detection {
  id: string;                 // uuid v4 — stable for canvas/list highlight
  classId: number;            // 0-6
  className: string;          // from metadata.classes[classId].name
  severity: 'critical' | 'high' | 'medium' | 'low';
  confidence: number;         // 0-1
  bbox: {
    x: number;                // top-left, original image space (px)
    y: number;
    w: number;
    h: number;
  };
  color: string;              // resolved hex from DESIGN.md severity scale
}
```

## C4. HistoryRecord Contract (IndexedDB / Dexie)

Stored client-side only. Schema version tracked in Dexie for future migrations.

```typescript
export interface HistoryRecord {
  id: string;                 // uuid v4 (primary key)
  createdAt: number;          // epoch ms (indexed)
  imageBlob: Blob;            // original image (jpeg, ~100-500KB after recompression)
  thumbnailDataUrl: string;   // 250x250 jpeg data URL for grid views
  detections: Detection[];    // C3
  inferenceMs: number;        // wall-clock time (preprocess + infer + postprocess)
  modelVersion: string;       // metadata.model.name (indexed)
  notes?: string;             // user-editable annotation
}

// Dexie schema v1
db.version(1).stores({
  records: 'id, createdAt, modelVersion, *detections.classId',
});
```

Export to CSV (T4.8) excludes `imageBlob` and `thumbnailDataUrl`. Columns: `id, createdAt, classes (joined), maxConfidence, inferenceMs, modelVersion`.

## C5. ModelStatus State Machine Contract

Exposed via `useModelStatus()` from `model-provider.tsx`. Consumed by `model-status-pill`, `model-loader-banner`, and `image-dropzone` (disable when not ready).

```typescript
export type ModelStatus =
  | { phase: 'idle' }
  | { phase: 'fetching'; loaded: number; total: number }
  | { phase: 'compiling' }
  | { phase: 'warming' }
  | { phase: 'ready'; source: 'network' | 'cache'; ep: 'webgpu' | 'wasm' }
  | { phase: 'error'; reason: string; retryable: boolean };
```

### Allowed transitions
```
idle ──► fetching ──► compiling ──► warming ──► ready
  │                                                │
  ├── (cache hit) ──► compiling ──► ready ─────────┘
  │
  └── (any phase) ──► error ──► (retry) ──► fetching
```

### Error reasons (string enum)
- `'network'` — fetch failed
- `'cors'` — HF Hub returned non-CORS response
- `'compile'` — ORT session creation failed (try WASM fallback)
- `'oom'` — out of memory (suggest samples-only mode)
- `'unknown'`

## C6. HF Spaces Gradio API Contract (Fallback)

Hosted at `huggingface.co/spaces/<user>/pipevision-yolo26m`. Used when ONNX path fails or as "server-side demo" comparison.

### Endpoint
- **Method**: `POST`
- **URL**: `https://<user>-pipevision-yolo26m.hf.space/run/predict`
- **Auth**: Public (no token)

### Request body
```json
{
  "data": [
    "data:image/jpeg;base64,...",
    0.25,
    0.45
  ]
}
```
- `data[0]`: image (base64 data URL)
- `data[1]`: conf threshold (float)
- `data[2]`: iou threshold (float)

### Response (200)
```json
{
  "data": [
    "data:image/png;base64,..."
  ],
  "duration": 5.42,
  "average_duration": 4.91
}
```
- `data[0]`: result image with bboxes drawn (base64 data URL)

### Errors
- `404` — Space not found / not public
- `503` — Space is sleeping (cold start ~30s, automatic wake on first request)
- `429` — Rate limit (HF community Spaces share quota)

### Frontend usage
```typescript
async function detectViaSpace(imageDataUrl: string, conf = 0.25, iou = 0.45) {
  const res = await fetch(`${HF_SPACE_URL}/run/predict`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ data: [imageDataUrl, conf, iou] }),
  });
  if (!res.ok) throw new Error(`Space error: ${res.status}`);
  const json = await res.json();
  return json.data[0] as string;
}
```

## Notes

- All contracts are **versioned implicitly** via Git history of this file. Breaking changes require coordinated updates to the producer and all consumers.
- C1 (model I/O) shape changes require model re-export and possible postprocess update.
- C4 (IndexedDB) schema changes require Dexie version bump + migration logic.
- C5 (ModelStatus) is the only contract with a state-machine constraint — adding phases requires updating all consumers.
